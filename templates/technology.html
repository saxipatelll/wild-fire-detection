HOG Features
Histogram of Oriented Gradients divides the image into cells, computes gradient magnitude and direction per pixel, then bins them into orientation histograms. Fire regions exhibit dense, multi-directional gradients — distinct from the uniform texture of sky or foliage.
# HOG config used in FireFound hog = cv2.HOGDescriptor( _winSize = (128, 128), _blockSize = (16, 16), _blockStride= (8, 8), _cellSize = (8, 8), _nbins = 9 ) # → ~3780 dimensional vector
SVM — RBF Kernel
Support Vector Machines find the maximum-margin hyperplane separating classes. The RBF (Radial Basis Function) kernel implicitly maps features to infinite-dimensional space, enabling non-linear boundaries — essential for the complex feature distributions of fire images.
Linear Fast, interpretable. Good baseline (~85–88%).
RBF Non-linear. Best accuracy (90%+). Tuned C & γ.
Poly Polynomial boundary. Slower, marginally worse.
PCA + StandardScaler
Raw HOG + HSV + Stat features produce a ~3924-dim vector per image. StandardScaler normalises each dimension to zero mean and unit variance — critical because SVM is sensitive to feature magnitude. PCA then reduces dimensionality by projecting onto the top principal components that explain 95% of variance (typically ~280–320 components). This removes noise, reduces training time, and often improves generalisation.
# StandardScaler: fit on train, transform both scaler = StandardScaler() X_train_s = scaler.fit_transform(X_train) X_test_s = scaler.transform(X_test) # PCA: keep 95% explained variance pca = PCA(n_components=0.95, svd_solver='full') X_train_p = pca.fit_transform(X_train_s) # ~3924 dims → ~300 dims
